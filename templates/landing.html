Task Manager API Documentation

Overview

A full-featured task management application with user authentication, task CRUD operations, filtering, and sorting capabilities.

Features

  • User registration and authentication with JWT tokens
  • Create, read, update, and delete tasks
  • Task filtering by priority and status
  • Task sorting by due date and priority
  • Task status management (pending/completed)
  • Priority levels (High, Medium, Low)
  • Due date tracking
  • Responsive web interface

Tech Stack

Backend

Python Flask

Database

SQLAlchemy ORM

Authentication

JWT (JSON Web Tokens)

Frontend

HTML, CSS, JavaScript

Security

Werkzeug Password Hashing

API

RESTful Web Services

API Routes

User Registration POST

Endpoint: /register

Description: Register a new user account

Request Body:

{ "username": "string", "email": "string", "password": "string (min 6 characters)" }

Response:

{ "message": "User registered successfully" }
User Login POST

Endpoint: /login

Description: Authenticate user and receive JWT token

Request Body:

{ "email": "string", "password": "string" }

Response:

{ "access_token": "jwt_token_string", "message": "Login successful" }
Note: All task-related endpoints require authentication. Include the JWT token in the Authorization header: Bearer <token>
Create Task POST

Endpoint: /tasks

Description: Create a new task

Request Body:

{ "title": "string (max 100 chars)", "description": "string (optional)", "due_date": "YYYY-MM-DD", "priority": "High|Medium|Low", "status": boolean }
Get All Tasks GET

Endpoint: /tasks

Description: Retrieve user's tasks with optional filtering and sorting

Query Parameters:

?priority=High|Medium|Low &status=true|false &sort_by=due_date|priority &order=asc|desc

Example:

GET /tasks?priority=High&status=false&sort_by=due_date&order=asc
Get Single Task GET

Endpoint: /tasks/<task_id>

Description: Retrieve a specific task by ID

Update Task PUT

Endpoint: /tasks/<task_id>

Description: Update an existing task

Request Body:

{ "title": "string (max 100 chars)", "description": "string (optional)", "due_date": "YYYY-MM-DD", "priority": "High|Medium|Low", "status": boolean }
Delete Task DELETE

Endpoint: /tasks/<task_id>

Description: Delete a specific task

Response Format

All API responses are in JSON format. Task objects have the following structure:

{ "id": integer, "title": "string", "description": "string", "due_date": "YYYY-MM-DD", "priority": "High|Medium|Low", "status": boolean, "created_at": "ISO timestamp", "user_id": integer }

Error Handling

Error responses include appropriate HTTP status codes and error messages:

{ "message": "Error description" }
  • 400 - Bad Request (validation errors)
  • 401 - Unauthorized (invalid/missing token)
  • 404 - Not Found (task doesn't exist)
  • 409 - Conflict (duplicate email during registration)
  • 500 - Internal Server Error
